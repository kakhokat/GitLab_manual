# 1. Создание личного репозитория с нужным .gitignore и простым README.MD

## Создание репозитория

Шаг 1. Зайдите в свой аккаунт на GitLab и нажмите на иконку "Groups" в верхней панели.

<img src="https://github.com/kakhokat/GitLab_manual/blob/main/1.1.png?raw=true" width=60%/>

Шаг 2. Затем перейдите во вкладку "Your group".

Шаг 3. Выберите команду с названием вашего учебного проекта.

<img src="https://github.com/kakhokat/GitLab_manual/blob/main/1.2.png?raw=true" width=60%/>

Шаг 4. Вы перешли на страницу своей команды. Теперь нужно создать репозиторий. Для этого нажмите на кнопку "New project".

<img src="https://github.com/kakhokat/GitLab_manual/blob/main/1.3.png?raw=true" width=60%/>

Шаг 5. Напишите название вашего репозитория и добавьте нужные данные. Готово!

<img src="https://github.com/kakhokat/GitLab_manual/blob/main/1.4.png?raw=true" width=60%/>

## Создание .gitignore и README.MD

На GitLab предусмотрена возможность создавать новые файлы. Для этого на странице репозитория нажмите на кнопку "+" и выберите "New file".

<img src="https://github.com/kakhokat/GitLab_manual/blob/main/1.5.png?raw=true" style="margin:auto;"/>

В открывшемся редакторе введите имя файла и нужное содержимое. Не забудьте нажать на кнопку Commit changes по завершении редактирования.

## .gitignore

.gitignore нужен чтобы указать в нем новые файлы, которые должны быть проигнорированы. Файлы .gitignore содержат шаблоны, которые сопоставляются с именами файлов в репозитории для определения необходимости игнорировать эти файлы.


## README.MD
README.MD это первый файл, который нужно читать, получив доступ к проекту на Github. Это текстовый файл, который распространяется вместе с программным обеспечением и содержит информацию о нём. Для редактирования файла понадобятся знание Markdown-разметки и базовое понимание HTML.
